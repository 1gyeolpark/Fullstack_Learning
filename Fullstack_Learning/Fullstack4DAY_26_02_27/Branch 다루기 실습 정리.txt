Branch 실습 정리

# 저장할 파일 만들기
파일 위치부분 클릭 - cmd
명령프롬프트 오픈된다

# 초안파일 생성
init.txt 생성후 아무내용 저장

# 해당 폴더를 저장소로 
git init

--------------------
# 초안파일 생성
init.txt 생성후 저장

# 초안 Commit
git add *
git commit -m "V0.0 INIT"
---------------------
# 확인
git log --oneline

---------------------
# Branch확인
git branch

# Branch 생성 (dev등 이름 자유)
git branch dev

# Branch 변경
git switch dev
or
git checkout dev
---------------------
# Dev Branch 작업 추가

# Dev's 작업 추가
git add *
git commit "V0.1 div_01 add"

# dev_02.txt 파일추가
git add *
git commit -m "V0.2 dev_02 add"

dev Branch 에서 작업 추가하고 저장해도 master는 아직 알 수 없음.

# 병합하기(Default: FF) (비효율. 커밋이 만들어지지 않음. 일단 학습)

git switch master
git merge dev

Merge기본옵션
(FF)Fast-Forwrd : 커밋 생성 x (지금 하는 것. 비권장)
non-Fast-Forward: 커밋 생성 o (많이 쓰는 것. 권장)

# 작업추가하기
git switch dev
dev_03.txt 추가하기

git add *
git commit -m "V0.3 dev_03 add"

# 병합하기 (Default: FF)

git switch master
git merge dev

# 병합하기 (non-FF)

git switch dev
dev_04.txt 파일 추가
git add *
git commit -m "V0.4 dev_04 add"

git switch master
git merge dev --no-ff
이렇게 메시지 안적으면 병합 커밋 메시지 적으라고 문서가 열린다.
저장-나가기
:wq 자동생성 메시지 그대로 사용.

dev 브랜치 위치를 master와 동기화. 
git switch dev
git merge master


# 병합하기 (non-FF)

git switch dev
dev_05.txt 파일 추가
git add *
git commit -m "V0.5 dev_05 add"

git switch master
git merge dev --no-ff -m "V0.5 merge"

git switch dev
git merge master

git log --oneline


non-FF부분 재학습


가지 Branch확인
git log --graph
